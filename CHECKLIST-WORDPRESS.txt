═══════════════════════════════════════════════════════════════════════════════
  TROUBLESHOOTING COMUM
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA: CSS não carrega
SOLUÇÃO:
☐ Verificar caminho do style.css no <link>
☐ Limpar cache do WordPress
☐ Verificar permissões do arquivo (644)

PROBLEMA: JavaScript não funciona
SOLUÇÃO:
☐ Verificar caminho do main.js no <script>
☐ Abrir console do navegador (F12) para ver erros
☐ Garantir que o script tem "defer" ou está no final

PROBLEMA: Links quebrados
SOLUÇÃO:
☐ Verificar se todos os links terminam em .html
☐ Verificar se os arquivos existem na mesma pasta
☐ Usar links relativos, não absolutos

PROBLEMA: Design quebrado no mobile
SOLUÇÃO:
☐ Verificar se viewport meta tag está presente
☐ Testar em modo responsivo do navegador
☐ Verificar se CSS está carregando

PROBLEMA: Páginas retornam 404
SOLUÇÃO:
☐ Ir em: Configurações → Links Permanentes
☐ Clicar em "Salvar Alterações" (regenera regras)
☐ Limpar cache
